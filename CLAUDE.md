# flying_buttress

flying_buttress is a software factory: a repository template and methodology for building applications end-to-end with AI-native workflows baked in from the start.

## Start here

- [ONBOARDING.md](ONBOARDING.md) — new to the repo? start here
- [MANUAL.md](MANUAL.md) — practical day-to-day reference for using the factory
- [buttress.md](buttress.md) — the manifesto: what flying_buttress is and why it exists
- [plan.md](plan.md) — the operating manual: how the factory runs on Claude Code
- [backlog.md](backlog.md) — open gaps and their status
- [docs/adr/README.md](docs/adr/README.md) — decisions made and work in progress

## Workflows

Skills are in [.claude/skills/](.claude/skills/). Available slash commands:

- `/spec` — generate a PRD for a new feature; enters plan mode before writing anything

## Repo structure

```
flying_buttress/
├── templates/        ← what gets stamped into every new sibling project
├── scripts/          ← scaffold.py and other factory tooling
├── .agents/          ← domain rules for this repo
├── .claude/skills/   ← slash command definitions
├── docs/adr/         ← architecture decision records
└── docs/specs/       ← specs generated by /spec
```
